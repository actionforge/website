Frequently asked questions

What exactly is Actionforge?

Actionforge offers an intuitive visual node interface to create and maintain complex GitHub Action workflows.

How do I create an Action Graph?

Using Actionforge, you can visually create your workflow via a VS Code Extension. Each node of your graph represents a GitHub Action. Together, they form what is called an 'Action Graph'. Once your graph is ready, you can commit it to your repository.

How do I execute an Action Graph as a GitHub Action workflow?

To execute an action runner on your GitHub Runners, you just need a traditional GitHub Actions workflow. The workflow file contains a single step that uses github.com/actionforge/action and points to your Action Graph - that's it.

Why is Actionforge not simply a UI wrapper for GitHub workflows?

Traditional GitHub workflows follow a linear execution order from top to bottom, which can be restrictive. In contrast, action graphs enable more complex and versatile node setups with different flow nodes like if-conditions, for-loops, switches and more while still being more secure.

Does Actionforge use cloud services?

Actionforge does not depend on external cloud services. An action graph is executed directly on your designated GitHub runners.

What is the pricing model for Actionforge?
